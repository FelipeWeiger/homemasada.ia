# Guia de Criacao de Ofertas Irresistiveis

> Baseado em "$100M Offers" de Alex Hormozi. Template para criar ofertas que parecem burras de recusar.

---

## 1. A Equacao de Valor

```
Valor = (Resultado Desejado x Probabilidade de Sucesso) / (Tempo x Esforco)
```

### Como Aumentar Valor:

| Variavel | Como Maximizar |
|----------|----------------|
| **Resultado Desejado** | O que o cliente REALMENTE quer? (nao o que voce vende) |
| **Probabilidade de Sucesso** | Garantias, casos de sucesso, provas |
| **Tempo** (diminuir) | Quanto mais rapido, mais valioso |
| **Esforco** (diminuir) | Quanto menos o cliente faz, melhor |

### Template - Preencha:

- **Resultado que o cliente quer:** ___
- **O que aumenta a chance de dar certo:** ___
- **Em quanto tempo entrega:** ___
- **O que o cliente precisa fazer:** ___

## 2. Identificando a Dor Real

> "Problema e generico. Dor doi no bolso."

### Perguntas Para Extrair a Dor:

1. O que acontece se continuar como esta?
2. Quanto isso custa por mes? (tempo, dinheiro, oportunidade)
3. Quem mais e afetado por esse problema?
4. Ha quanto tempo convive com isso?
5. O que ja tentou fazer para resolver?

## 5. Precificacao Value-Based

### Formula Base:

```
Valor Entregue / 10 = Preco Minimo
```

Se sua solucao economiza R$30.000/ano, preco minimo = R$3.000.

### Template - Preencha:

| Metrica | Valor |
|---------|-------|
| Economia/ganho anual do cliente | R$ ___ |
| Preco minimo (/ 10) | R$ ___ |
| Seu preco proposto | R$ ___ |
| Multiplo de valor (economia / preco) | ___x |

**Regra:** Se multiplo < 3x, preco esta alto ou valor nao esta claro.

### Modelo de Cobranca:

- [ ] Setup + Recorrente (implementacao complexa, valor continuo)
- [ ] Projeto Fechado (entrega unica, valor imediato)
- [ ] So Recorrente (baixa barreira de entrada)
- [ ] Performance (so paga se funcionar)

## 6. Scripts de Prospecao

### 6.1 Abertura (ACA: Acknowledge, Compliment, Ask)

```
Oi [nome], vi que voce [algo especifico sobre ele/empresa].

[Elogio genuino, nao generico].

[Pergunta simples que qualifica]:
- "Voces ainda [dor em forma de pergunta]?"
```

**Sua abertura:** ___

### 6.2 Pergunta de Qualificacao Hormozi

```
Ainda ta [dor em 4-5 palavras]?
```

Exemplos:
- "Ainda ta perdendo leads por demora no WhatsApp?"
- "Ainda ta buscando automatizar seu atendimento?"

**Sua pergunta:** ___

### 6.3 Follow-ups

**Follow-up 1 (3 dias):**
```
[Nome], so checando se viu minha mensagem.
Se nao faz sentido agora, sem problema. So me avisa.
```

**Follow-up 2 (7 dias):**
```
Ultima mensagem sobre isso, [Nome].
Se [dor] nao e um problema pra voces, otimo - ignore.
Se for, ainda to aqui.
```

**Regra:** 3 follow-ups, depois para. Nao seja chato.

### 6.4 Objecoes Comuns

| Objecao | Resposta |
|---------|----------|
| "E caro" | "Quanto custa continuar com [dor]? So esse custo ja paga o sistema." |
| "Nao tenho tempo" | "A implementacao e nossa. Voce so precisa [minimo necessario]." |
| "Preciso pensar" | "Faz sentido. O que especificamente voce precisa avaliar?" |
| "Ja tentei algo parecido" | "O que deu errado? [Escuta] Entendi. A diferenca aqui e [diferencial]." |

## 7. Checklist Final

Antes de prospectar, marque tudo:

- [ ] Dor quantificada em R$/mes
- [ ] ICP especifico (nao generico)
- [ ] Garantia que elimina risco do cliente
- [ ] Preco < 10x do valor percebido
- [ ] Lista de 50 leads pronta
- [ ] Demo ou prova de conceito gravada
- [ ] Scripts de abertura escritos
- [ ] Objecoes mapeadas com respostas

**Se falta algo:** Complete antes de prospectar.
**Se tudo marcado:** Para de preparar e manda mensagem.

## Lembrete Hormozi

> "Esse guia nao e pra ler. E pra preencher e mandar 50 mensagens. Se voce leu e nao preencheu, e procrastinacao com nome bonito."

**Pergunta obrigatoria depois de preencher:**
"Quantas mensagens vou mandar HOJE?"

Se a resposta for zero, voce nao terminou - voce fugiu.
